## Архитектурные консультации

### Базы данных
Выбор модели хранения данных (реляционные, NoSQL, гибридные, time‑series, графовые и т.д.), проектирование схемы, шардирование, репликация, стратегии резервного копирования и восстановления после аварии, оптимизация запросов, индексов, кэширования (CQRS, Event‑Sourcing, multi‑level cache), оценка операционных затрат и производительности.

### Брокеры сообщений
Сравнение Kafka, RabbitMQ, Pulsar, NATS, ActiveMQ и других платформ; проектирование топиков/очередей, маршрутизации, стратегий повторных попыток (retry, dead‑letter, экспоненциальная задержка); конфигурация семантики доставки (at‑most‑once, at‑least‑once, exactly‑once); мониторинг задержек, пропускной способности и долговечности сообщений.

### Системный дизайн
Построение архитектуры по методологии C4 (концептуальная, контейнерная, компонентная, кодовая); определение границ контекстов DDD и моделирование бизнес‑логики; стратегии масштабирования (горизонтальное/вертикальное, stateless сервисы, кэширование на границе); отказоустойчивость (резервирование, circuit‑breakers, graceful degradation); наблюдаемость (логирование, метрики, трассировка с использованием OpenTelemetry).

### Микросервисы
Определение «один сервис — одна бизнес‑возможность» и проектирование bounded context; выбор API Gateway / BFF, service‑mesh (Istio/Linkerd); проектирование схем конвейера CI/CD, контейнеризации (Docker) и оркестрации (Kubernetes); подходы к версии API и стратегии совместной работы.

### Интеграция и коммуникация
Синхронный vs асинхронный обмен данными, паттерны публикации/подписки, request/reply; проектирование схем событий, контрактов сообщений (protobuf, Avro, JSON Schema).

### Безопасность и соответствие
Архитектура аутентификации/авторизации (OAuth2, JWT, mTLS); шифрование данных в состоянии покоя и при передаче, хранение секретов; концепции «безопасности по умолчанию» и «надежной безопасности».

### Миграции и эволюция
Планы поэтапной миграции (например, рефакторинг моноэта к микросервисам); оценка рисков, бенчмаркинг производительности, стратегии отката.
